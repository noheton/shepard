# @dlr-shepard/v2-client (TypeScript, Kiota-generated)

TypeScript client for shepard's `/v2/` API surface, generated by
[Microsoft Kiota](https://learn.microsoft.com/openapi/kiota/overview)
from the per-shelf OpenAPI document `/shepard/doc/openapi/v2.json`
(P4c, shipped).

This is the **CG1a** new-baseline client per
[ADR-0022](../../aidocs/63-architecture-decision-log.md#adr-0022). The
sibling `clients/` directory holds the legacy `/shepard/api/...` clients
emitted by OpenAPI Generator (CG1b).

> **Frontend note.** This package is the *Node / Deno / Bun*
> client — for the Nuxt 3 frontend's TanStack-Query layer the
> tactical secondary is Hey API per `aidocs/57 §4.3`. That CG1e
> slice is deferred.

## Install

```bash
npm install @dlr-shepard/v2-client
```

(Published on npm by CG1c — until then, build locally from this folder.)

## Quick start

```ts
import { AnonymousAuthenticationProvider } from "@microsoft/kiota-abstractions";
import { FetchRequestAdapter } from "@microsoft/kiota-http-fetchlibrary";
import { createShepardV2Client } from "@dlr-shepard/v2-client";

const auth = new AnonymousAuthenticationProvider();   // or Bearer / X-API-Key
const adapter = new FetchRequestAdapter(auth);
adapter.baseUrl = "https://shepard.example.com";
const client = createShepardV2Client(adapter);

// Fluent path-builder mirrors the URL:
// /v2/dataobjects/{appId} -> client.v2.dataobjects.byAppId(appId).get()
```

## Regenerating

```bash
# Boot shepard locally so /shepard/doc/openapi/v2.json is reachable, then:
make -C .. generate-typescript
```

Pinned Kiota version: see `clients-v2/Makefile` (`KIOTA_VERSION`).
