## Summary for funders and programme managers

Shepard v6.0.0-rc.1 ships a production-grade six-substrate research data
platform — Neo4j (provenance graph), PostgreSQL + TimescaleDB (relational and
high-rate timeseries), MongoDB (unstructured payload index), PostGIS (spatial
data), and Garage S3-compatible object storage — replacing the upstream
single-substrate architecture with a unified stack that handles any research
data kind from a single API surface.
Every write — whether authored by a researcher, an automated pipeline, or an
AI co-pilot — is recorded as a typed `:Activity` node carrying actor, timestamp,
source mode (`human | ai | collaborative`), and a tamper-evident HMAC audit
chain, fulfilling the provenance traceability requirements of EU Horizon Europe
DMPs and Helmholtz RDM mandates.
MFFD thermoplastic CFRP manufacturing data from ZLP Augsburg is now ingestible
end-to-end: the v16 PRESERVE-HIERARCHY importer replicates full DataObject trees
across instances, the COPY-protocol bulk ingest endpoint handles high-rate sensor
channels at 3–5× upstream throughput, and the MFFD digital thread — AFP layup →
welding → NDT → rework — is representable as a traceable Predecessor/Successor
provenance chain.

## What shipped in this RC

### Infrastructure and health
- **Six-substrate stack** — Neo4j + PG/TimescaleDB + MongoDB + PostGIS + Garage S3 all connected with per-DB health separation, graceful degradation (RFC 7807 503 + `Retry-After`), and automated recovery scheduling (A1a–A1f, P1)
- **Fail-fast migrations** — `MigrationsRunner` propagates `MigrationsException` on startup; Flyway ceiling at 10 retries × 5 s; `neo4j-migration-chain-readiness` SmallRye healthcheck validates every classpath migration's checksum against the live DB
- **Migration progress endpoint** — `GET /migrations/progress` (P3)
- **Garage S3 object storage** — replaces MinIO (community edition retired); ADR-0024

### Security and auth
- **Instance-admin role** — `instance-admin` JWT claim + Neo4j `:HAS_ROLE` dual-source; `@RolesAllowed` works; bootstrap-token mechanism; orphan-permissions fail-closed (A0, C3)
- **API-key roles and expiry** — `validUntil` field + JWT `exp`; `roles` claim with allowlist (L5, A0)
- **Cypher injection hardening** — parameterised queries across all DAOs; property-name allowlist (C5, C5b)
- **RFC 7807 error responses** — `application/problem+json` with trace-ID sanitisation; legacy `ApiError` preserved for upstream-client compat (H4)
- **Permission audit log** — Postgres `permission_audit_log` table; `GET /v2/admin/permission-audit/log` with filters + frontend pane (F3)
- **JWT `iat`-keyed permission cache** — prevents stale cache hits across JWT rotations (F4)
- **Fail-closed Neo4j guard** — `PermissionsService.isAllowed()` returns `false` immediately when Neo4j is down (F5)

### Identifiers and API surface
- **UUID v7 `appId`** on all 28 Neo4j labels; backfill migration; read-path switched to `appId` (L2a–L2c)
- **`/v2/` API shelf** — collections + DataObjects CRUD; batch create (207 Multi-Status, 1–500 items); DataObject detail with typed container lists + ancestor/descendant chain endpoints; OpenAPI split at `/shepard/doc/openapi/{v1,v2}.json` (L2d, MFFD-BATCH-01, P4, P4c)
- **SHACL validator** — `POST /v2/shapes/validate` with Apache Jena; HMAC audit chain on `:Activity` nodes (SHACL-1, SHACL-1b)

### Timeseries
- **Single-key channel identity** — `shepard_id` UUID column on `timeseries` table; `TsChannelResolver`; 5-tuple moved to `channel_metadata` side table; `GET /v2/timeseries-containers/{id}/channels` exposes `shepardId` on wire (TS-ID PR-1, PR-2, PR-3, TS-CORE-SCHEMA-01)
- **High-throughput ingest** — COPY-protocol bulk ingest endpoint (3–5× VALUES INSERT); multi-channel bulk fetch; LTTB downsampling with TimescaleDB pre-aggregation guard (TS-OPT1–3)
- **Channel semantic dual-write** — metadata fields written to Neo4j `SemanticAnnotation` nodes in parallel (TS-SEMANTIC-01)

### Import and provenance
- **v16 PRESERVE-HIERARCHY importer** — 3-pass tree replication; fan-out workers; handles MFFD collection trees of ~8 500 DataObjects (v16)
- **Provenance HMAC audit chain** — per-`Activity` `HMAC-SHA256` stamp with chained `prevHmac`; tamper-evident; best-effort (SHACL-1b)
- **Admin activity log** — `GET /v2/provenance/activities`; `AdminActivityLogPane.vue` (GH #68)
- **Payload version history** — SHA-256 + version counter on file uploads; frontend history dialog (PV1a/PV1b)

### Developer experience and operations
- **Smart warmup + smoke test make targets** — `make redeploy-*` chains wait-for-health + smoke; `make wait-for-health` blocks until all six substrates are up
- **Runtime feature toggles** — `GET/PATCH /v2/admin/features` with `source` field (`default/config/runtime`) (A3b, DX7)
- **Plugin admin** — `GET/PATCH /v2/admin/plugins`; `PluginsAdminPane.vue` (PM1a–PM1e)
- **OpenAPI shelf split** — v1 and v2 specs served separately; combined spec untouched for upstream-client compat
