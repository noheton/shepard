# v5 source — OpenAPI 5.4.0 summary (legacy-compat ground truth)

The **authoritative wire contract** for everything that lives under
`/shepard/api/...` on this fork. This is the *upstream*
`gitlab.com/dlr-shepard/shepard` 5.4.0 spec — the surface a real
`dlr-shepard` deployment (e.g. DLR cube3 at
`backend.bt-au-cube3.intra.dlr.de`) exposes today.

**Spec file:** `backend/src/test/resources/fixtures/v5/openapi-5.4.0.json` (full spec, 282 KB, 90 paths, 151 endpoints, 89 schemas)

**Why this matters for the fork:**
- CLAUDE.md API-version policy: *"The upstream API surface stays frozen.
  `/shepard/api/...` paths must remain byte-compatible with
  `gitlab.com/dlr-shepard/shepard 5.2.0`."* (5.4.0 is the most recent
  upstream snapshot we have on file; the 5.2.0→5.4.0 delta is itself
  additive within v1 — see `aidocs/integrations/93-mffd-import-v15-requirements.md`.)
- Every PR touching `/shepard/api/...` is gated by `V1WireFidelityIT`
  against fixtures in `backend/src/test/resources/fixtures/v5/`. This
  OpenAPI is the **source of truth** for what those fixtures must
  match.
- `shepard-plugin-v1-compat` (Phase 1 shipped, Phase 2 designed) carries
  the byte-stability commitment. Its acceptance criteria are
  "every response in 5.4.0 OpenAPI returns byte-identical bytes from
  this fork." Without this spec, no acceptance test.

## How to use this when working legacy-compat

Recipes:

1. **Adding a `/shepard/api/...` endpoint that already exists in 5.4.0**
   → fork must serve byte-identical response shape. Look up the
   `responses[*].content."application/json".schema` block in the spec,
   compare against this fork's response IO class field-for-field. Any
   delta is a fork drift bug.
2. **Adding a new endpoint that does NOT exist in 5.4.0** → MUST land
   under `/v2/...`, never under `/shepard/api/...`. Period.
3. **Renaming a field on a `/shepard/api/...` response** → forbidden.
   Add the new field on the `/v2/...` companion endpoint; leave v1 alone.
4. **Confirming an upstream OpenAPI matches reality** → boot a
   `dlr-shepard:5.4.0` container, `curl http://localhost:8080/shepard/doc/openapi`,
   diff against this JSON. (Done last when we built the corpus per
   task #128.)
5. **Schemas to know cold** — `Collection`, `DataObject`, `Permissions`,
   `Roles`, `Timeseries`, `FileReference`, `ShepardFile`, `SearchBody`,
   `SearchScope`, `BasicEntity`. These appear in nearly every response
   shape and are the most-touched compat surfaces.
