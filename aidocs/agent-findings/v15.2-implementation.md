## What I found

The 2700-line `mffd-import-v15.py` monolith ships with a `warmup()`
that only does `/v2/users/me` on the dest + `/shepard/api/collections`
on the source. By the time a 10-hour cross-instance import discovers
a missing scope or a wire-shape drift, hours of forging work are
already on the floor. The user's directive on 2026-05-22 ("script
should be smart and have a warmup / probing phase everything works
(write access to v5 is enabled for testing)" + "unexpected replies
lead to abort and actionable diagnostic report") is the right
intervention — a strict pre-flight that exercises every payload kind
the run will touch.

The canonical wire-contract source is now committed in-tree at
`backend/src/test/resources/fixtures/v5/openapi-5.4.0.json` (282 KB,
90 paths, 89 schemas). IMPORT-W3's comparator reads that JSON
directly — no network access, no fragile upstream URL.

## What shipped

### `_smart_warmup.py` (new module, ~860 LOC)

Co-located with the main script (not under `tools/`) because the
comparator has exactly one Python consumer today — keeping it
adjacent minimises path indirection.

- **`WarmupReport`** — structured diagnostic with append-only probe
  trace. `to_text()` is the operator-readable block (one fact per
  line, key-value aligned, matches the `diagnostic-script-to-upload`
  pattern); `to_json()` round-trips for machine consumption.
- **`WarmupAborted`** — carries the report + exit code; raised on
  first failure.
- **`SmartWarmup`** — orchestrator. Probe order: auth (dest) → auth
  (source) → read (dest) → read (source) → write (source) → write
  (dest) → Garage. Each probe runs the OpenAPI shape comparator
  against the response body. Cleanup of throwaway DataObjects runs
  from `finally`, regardless of success or abort.
- **`compare_against_openapi`** — IMPORT-W3's helper. Walks
  `paths[...].<method>.responses[<status>].content."application/json".schema`,
  resolves `$ref` + `allOf` one level deep, returns a `ShapeDiff`
  with separate lists for missing-required / type-mismatch /
  extra-unknown. Gracefully `skipped=True` when the spec lacks an
  endpoint — the comparator never refuses to run.
- **Exit-code constants** match `feedback_warmup_fail_fast_diagnostic.md`
  exactly: 2 (auth), 3 (source-unreachable), 4 (garage-down),
  5 (operator-interrupt), 6 (wire-shape-drift), 7 (write-permission-denied).
  A test asserts each constant's literal value.

### Integration into `mffd-import-v15.py` (47 LOC delta, additive)

- Best-effort import (`try / except ImportError`) so a missing
  module degrades to legacy warmup transparently.
- `--smart-warmup` (default on) and `--legacy-warmup` (opt-out)
  argparse flags.
- Single integration point right before `dest_client.warmup()`. On
  `WarmupAborted` the script prints the structured block to stderr
  and exits with the per-failure-class code.
- The legacy `dest_client.warmup()` and `warmup_probe_and_gate()`
  still run after smart warmup succeeds — smart warmup is additive
  pre-flight, not a replacement.

## OpenAPI coverage on the v5.4.0 spec

Quantification per the acceptance criteria:

| Endpoint | Method | Spec schema? | Notes |
|---|---|---|---|
| `/users/{username}` | GET | yes | Source-auth probe — full required-fields check |
| `/collections` | GET | yes | Read probe (both sides) — array of `Collection` |
| `/collections/{collectionId}/dataObjects` | POST | yes | Source-write + dest-write probe |
| `/v2/users/me` | GET | no | Fork-only `/v2` surface — spec is upstream-only; probe runs without diff (spec-skipped) |
| `/v2/file-containers/{appId}/upload-url` | POST | no | Fork-only Garage hook; existing `garage_preflight()` does the diagnostic |

**3/3 of the probed v1 endpoints have schemas.** The two
spec-skipped endpoints (`/v2/users/me`, Garage upload-url) are both
fork-only surfaces — they correctly fall through the comparator's
"skipped" path with no false-positive failure.

## Wire-shape surprises

One I want to flag, because it's the kind of thing the comparator
exists to catch:

- The spec's `Collection.required` list is broad: `id`, `createdAt`,
  `createdBy`, `updatedAt`, `updatedBy`, `name`, `dataObjectIds`,
  `incomingIds` — eight required fields. A real `/collections`
  response that omits any of `createdBy` / `updatedBy` / `incomingIds`
  (legitimately null cases) is currently flagged as a violation by
  the comparator because the spec declares them required but
  `nullable: true`. The comparator's `_check_type` returns OK for
  nullable+null, but **a missing key** (not `null`, absent) still
  flags `missing_required_fields`. This is the right behaviour per
  the OpenAPI spec — `required` means present-with-value-possibly-null —
  but worth keeping in mind during the first live run. If a real
  source emits `updatedBy: null` it's fine; if it omits the key
  entirely the warmup will abort with exit 6 and the operator gets
  the diff verbatim.

## Deferred items

These are explicitly out of scope for this PR (called out in `aidocs/16`):

- **Main-loop instrumentation** — the IMPORT-W2 structured diagnostic
  shape only applies to the warmup phase today. Bringing the same
  `WarmupReport`-style shape to mid-run failures (e.g. a 5xx on the
  500th DO) is a separate slice.
- **JVM-side parallel comparator** — IMPORT-W3 mentions
  `V1WireFidelityIT` should consume the same spec from the JVM side.
  Java implementation deferred — the Python script is the only
  current consumer.
- **`--smart-warmup` write probes against the dest collection at
  current point** — at the integration site we have the SHEPARD_URL
  + dest client but the dest collection app-id is resolved later by
  `ensure_dest_collection()`. So write_to_dest=False for now; we
  could re-run a partial warmup post-collection-create to close
  that gap. Tracked as a follow-up note in the orchestrator's
  docstring.
- **Operator interrupt (exit 5)** — exit code reserved + asserted in
  tests but no current code path emits it. The orchestrator runs
  synchronously; SIGINT propagates as a normal KeyboardInterrupt and
  the cleanup `finally` still runs. A future enhancement would
  install a SIGINT handler that emits a clean exit-5 with the
  partial probe trace.

## Exit-code matrix

| Code | Class | Surface |
|---|---|---|
| 0 | OK | warmup completed clean; main loop runs |
| 1 | generic | pre-existing v15 fallback (missing license, lock conflict) |
| 2 | auth | JWT expired / API key revoked / 401 on auth probe |
| 3 | source-unreachable | DNS / TLS / connection failure |
| 4 | garage-down | upload-url returns 503/404 or `garage_preflight` raises |
| 5 | operator-interrupt | reserved (SIGINT) — not yet emitted |
| 6 | wire-shape-drift | comparator flags missing required / type mismatch |
| 7 | write-permission-denied | 401/403 on POST during write probe |
