### 2. UI21-SIZEBAR-DATA (S)

**Row:** Richer sizebar metric per container kind on the `/containers` list — channel count for TS, file count for File, entry count for StructuredData.

**Branch:** `UI21-SIZEBAR-DATA-per-kind-cardinality`
**Status:** dispatched — PR #1764

**Why:** The current sizebar uses CC1e referenced-by-collection count as a proxy. That number tells a user nothing about how much data is in the container. Per-kind cardinality is the meaningful metric. TimeseriesContainer already had a `/stats` endpoint (TS_STATS1); this parity-fills FileContainer and StructuredDataContainer.

**What shipped:** Two new backend stats endpoints (`GET /v2/file-containers/{id}/stats` → `{ fileCount }`, `GET /v2/structured-data-containers/{id}/stats` → `{ entryCount }`). New frontend composable `useContainerCardinality.ts` + `ContainerCardinalityCell.vue`. `ContainerList.vue` sizebar now reads domain cardinality. Frontend typecheck and lint pass; 1053/1058 unit tests pass (5 pre-existing failures unrelated to this PR).
