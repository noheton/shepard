# Runbook — v15.15 + completeness v1.1 hot-deploy (2026-05-23)

**Goal**: Roll cube to v15.15 (predefined SD containers) + completeness-check v1.1 (container-shape sprawl warning) and restart importer with the env vars the predefined shape requires.

**Prerequisite**: Importer stopped on cube (`pkill -f mffd-runner.sh` or its tmux is broken). Predefined SD containers already created in dest by the dev box (IDs 645000 + 645003).

**Expected duration**: ~3 min. Cube-side only.

---

## Step 1 — Hot-fetch v15.15 + completeness v1.1 + watchdog (on cube)

One curl block. Run as `cube` user from `/home/cube/mffd-export`:

```bash
cd /home/cube/mffd-export && \
curl -fL -o /tmp/mffd-import-v15.py        https://raw.githubusercontent.com/noheton/shepard/main/examples/mffd-showcase/scripts/mffd-import-v15.py && \
curl -fL -o /tmp/mffd-completeness-check.py https://raw.githubusercontent.com/noheton/shepard/main/examples/mffd-showcase/scripts/mffd-completeness-check.py && \
curl -fL -o /tmp/mffd-import-health.sh     https://raw.githubusercontent.com/noheton/shepard/main/examples/mffd-showcase/scripts/mffd-import-health.sh && \
echo "=== expected sha256:" && \
echo "b4c6db28998e6e71edbe85a5a5ec3ab6b4cf717346bf54d0887d4a3fda72596f  mffd-import-v15.py (v15.15, BUG-F2 predefined SD)" && \
echo "9601d01b53f41fc9916dcc6238f5b0e05e3737c7e394a161534e1f58584087b8  mffd-completeness-check.py (v1.1, container-shape sprawl warning)" && \
echo "1855a752bee24184009bb23101d98ec86a02e823ef0e1a381301d53e55c51ac8  mffd-import-health.sh (v1.1)" && \
echo "=== actual sha256:" && \
sha256sum /tmp/mffd-import-v15.py /tmp/mffd-completeness-check.py /tmp/mffd-import-health.sh && \
grep '^IMPORT_SCRIPT_VERSION' /tmp/mffd-import-v15.py && \
head -2 /tmp/mffd-completeness-check.py | tail -1 && \
cp mffd-import-v15.py mffd-import-v15.py.bug-f2.bak 2>/dev/null || true && \
cp /tmp/mffd-import-v15.py mffd-import-v15.py && \
cp /tmp/mffd-completeness-check.py mffd-completeness-check.py && \
cp /tmp/mffd-import-health.sh mffd-import-health.sh && \
chmod +x mffd-completeness-check.py mffd-import-health.sh && \
rm -f .mffd-import.lock && \
echo "v15.15 + completeness v1.1 staged. Set env vars (step 2) before relaunch."
```

**Expected stdout (key lines)**:
- `IMPORT_SCRIPT_VERSION = "15.15"`
- sha256 lines match: `fc8250c5…` → `b4c6db28…` (v15.15), `…` → `9601d01b…` (v1.1)
- `"""mffd-completeness-check.py — post-migration completeness verifier (v1.1)`

If any sha256 mismatches: STOP, do not proceed. The repo may not have pushed to `main` yet; check `git log -3 origin/main` on this dev box.

## Step 3 — Reset SD-related state entries (on cube)

State file tracks per-DO "sd already done" markers. Reset only those entries so the SD work re-runs against the new predefined containers; file + TS state preserved:

```bash
cd /home/cube/mffd-export
cp mffd-import-state.json mffd-import-state.json.bak-pre-v15.15-$(date +%Y%m%d_%H%M%S)
python3 <<'PYEOF'
import json
with open('mffd-import-state.json') as f: s = json.load(f)
before = len(s.get('structured_done') or [])
s['structured_done'] = []
with open('mffd-import-state.json', 'w') as f: json.dump(s, f, indent=2)
print(f'reset {before} structured_done entries; file + TS state preserved')
PYEOF
```

## Step 5 — Run completeness check after import completes (on cube)

```bash
cd /home/cube/mffd-export && ./mffd-completeness-check.py
```

**Expected stdout key lines**:

- `[shape-ok]` if there's exactly 1 MFFD SD container per step and 1 MFFD TS container per step (this is the new v15.15 invariant).
- `[sprawl-warning]` followed by container ids if duplicates exist (e.g. the 3 historic tapelaying TS containers 528098 / 589657 / 599903 we identified today — those will be flagged until consolidated via TSDB-DDL-1).
- `SUMMARY: {kind: mffd-completeness, exit: 0|1|2|3, ...}` final one-liner.

Exit codes: `0` PASS, `1` PARTIAL, `2` MISSING-DOS, `3` CONN-FAIL.

The report writes to:
- `mffd-completeness-<session>.json`
- `mffd-completeness-<session>.md`
- Uploaded as a `MIGRATION-COMPLETENESS-<session>` DataObject in the dest collection.

If v15.15 misbehaves, restore the v15.14 binary you backed up in Step 1:

```bash
cd /home/cube/mffd-export && \
pkill -f mffd-runner.sh ; \
cp mffd-import-v15.py.bug-f2.bak mffd-import-v15.py && \
unset MFFD_SD_CONTAINER_TAPELAYING MFFD_SD_CONTAINER_BRIDGEWELDING && \
```

The 2 predefined SD containers 645000 + 645003 remain in dest (they're empty until v15.15 fills them); they're not a problem for v15.14, just unused.
